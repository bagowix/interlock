47@runtime_checkable 

48class Storage(Protocol): 

49 """Coordinates breaker state across instances through a shared backend. 

50 

51 Every operation is an atomic *intent*, not a raw read-modify-write: the 

52 backend owns the atomicity (e.g. a Lua script or a locked section), so racing 

53 instances never desync. The methods carry mechanism only — threshold policy 

54 stays in the core state machine, which calls ``trip_open`` / ``close`` once 

55 it has decided. ``ttl`` refreshes the key's expiry so abandoned instances let 

56 state self-expire. 

57 

58 This is the synchronous contract; ``AsyncStorage`` mirrors it for async 

59 breakers. No method may raise into the protected path — the engine degrades 

60 to local state on backend failure. 

61 """ 

62 

63 def read(self, name: str) -> SharedState | None: 

64 """Return the current shared view, or ``None`` if no key exists.""" 

65 ... 

66 

67 def trip_open( 

68 self, *, name: str, ttl: float, expected_version: int | None = None 

69 ) -> SharedState: 

70 """Transition to ``OPEN``, stamping the backend's time as ``opened_at``. 

71 

72 Idempotent while already ``OPEN`` (the first opener's time stands); 

73 from ``HALF_OPEN`` it reopens with a fresh ``opened_at`` and clears 

74 probe accounting. With ``expected_version`` set, the transition is 

75 fenced: it applies only while the backend still holds that version, so 

76 a decision computed from a stale view cannot clobber a newer state. 

77 A fenced-out call is a no-op returning the current view. 

78 """

106 ``probes_completed >= probes_permitted`` as "round finished, decide now". 

107 """
